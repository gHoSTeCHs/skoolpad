Block 3.3 · intermediate

Algebraic simplification before substituting

When direct substitution gives you 0/0, you don't give up — you reshape the expression until the singularity disappears.

The first thing every student tries when computing a limit is to simply plug in the value. Most of the time it works, and the limit is just the value of the function at that point. But occasionally substitution returns the indeterminate form 0/0 — and that’s where algebra earns its keep.

Consider the limit

limx→2 (x² − 4) / (x − 2)

Direct substitution gives 0/0, which tells us nothing. But the numerator factors as (x−2)(x+2), so we can cancel the troublesome (x−2) term:

= limx→2 (x + 2) = 4

Why this is allowed

Cancelling (x−2) changes the function at x = 2, but limits don’t care about the value at the point — only what’s happening on the way there. Since the simplified function agrees with the original everywhere except at x = 2, both functions have the same limit.

The pattern

Whenever you hit 0/0 with a rational function, ask: can I factor either piece? The cancellation almost always reveals the limit. For exam-style WAEC and JAMB problems, this single trick handles a surprising share of the questions.

Design moves demonstrated
0. Layout breakout
Show pages opt out of AdminLayout and render in a new ContentStudioLayout. Index and create pages keep the admin sidebar. An "← Admin" affordance in the top bar returns to the shell. Zero backend change.
1. Vertical stage rail
56px wide. Replaces the horizontal 5-step stepper, frees ~64px vertical, always accessible. Tooltip on hover gives context without taking space.
2. 56px top bar
Project breadcrumb · status pill · single global model chip · log access. Replaces the "AI Routing" mega-card that previously restated stage models four times.
3. Three-column persistent layout
Topics · Blocks · Editor — all visible at once. No drilldown navigation. Switch topics in one click, like an email client or IDE file tree.
4. Document-first editor surface
Manuscript column at 680px max-width with paper grain background. Bricolage Grotesque for the block title. Floating action bar with Approve as the primary action.
5. Compound generate button
Model name lives inside the action. Picking a different model = one click into the chevron. Three-mode "default / stage / run" radio collapsed into a single quiet "save as default" link.
6. Generation as focal moment
Workspace dims, centered card streams structured progress events with per-block timing. Honey accent identifies AI moments. "Run in background" lets you keep editing.
7. Status as filled pills
Decisive colored pills replace the mono-uppercase status text that drained hierarchy. Block list shows status as a colored dot with breathing-room pulse for the active generation.
8. Inspector peek (right rail)
Guidance, metadata, drift advisory, history each get an icon tab. Click to slide open over the editor. Default state is closed — the editor gets the room.
9. Typography hierarchy reset
Mono-uppercase reserved for actual technical strings (model_ids, paths, token counts, durations). Section labels in sentence-case sans semibold — readable, not a whisper.
10. Honey palette claims AI moments
The existing but underused Honey accent now consistently signals "this is an AI generation event" — pulse on generating blocks, shimmer on the focal card, accent on the run-override button.
