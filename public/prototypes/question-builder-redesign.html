B I U H2 H3 • 1. ∑ { } 214 words · 1 illustration · editing

The LRU algorithm tracks every page reference and, on a page fault when memory is full, evicts the page whose most recent access is furthest in the past. The intuition is locality: pages used recently are likely to be used again soon, while pages untouched for a long stretch are the safest to discard.

Two implementation styles exist:

Counter-based: attach a "last access" timestamp to each page-table entry. On every reference, write the current clock to that entry. On eviction, scan for the smallest timestamp.

Stack-based: maintain a doubly-linked list of pages ordered by recency. On reference, move the page to the head. On eviction, remove the tail.

Illustration · stack-based LRU SVG · 320 × 180
Caption · The recency stack model: each reference moves the page to the head; the tail is always the LRU candidate.

In practice, true LRU is expensive in hardware, so OSes approximate it (e.g., the accessed bit clock algorithm).
