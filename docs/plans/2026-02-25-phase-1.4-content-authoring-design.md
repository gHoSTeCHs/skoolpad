# Phase 1.4: Admin — Content Authoring Design

**Date:** 2026-02-25
**Status:** Approved
**Depends On:** Phase 1.3 (Admin Education Systems — completed)
**Reference:** Architecture Redesign/05-Revised-Phase-1-Brief.md

---

## Context

Phase 1.3 is complete: education systems, tiers, levels, streams, assessment types, grading scales, and institution types are all manageable via admin CRUD. Topics, courses, course-topic mappings, department offerings, and bulk import are already built from old Phase 1.3.

Phase 1.4 adds three new features on top of the existing foundation:

1. **Content Blocks Tree Editor** — hierarchical block structure under each canonical topic
2. **Course-Block Mappings** — map specific content blocks to courses with teaching depth and scheduling
3. **Scheme of Work Manager** — weekly teaching calendar for secondary school curriculum subjects

---

## Feature 1: Content Blocks Tree Editor

**Page:** `/admin/topics/{topic}/blocks`
**Controller:** `ContentBlockController`
**Entry point:** "Manage Blocks" dropdown menu item on topics index + button on topic edit page

### Layout

Two-panel split (inspired by the architecture showcase's Interactive Block Explorer):

- **Left panel (60%):** Tree view of blocks with expand/collapse, drag-and-drop reorder via dnd-kit
  - Each node shows: path number, block type icon, title, read time, difficulty badge
  - "Add Block" button at root level + "Add Child" on each container node
  - "..." menu per node: Edit, Delete, Add Child, Toggle Publish
- **Right panel (40%):** Detail/edit form for selected block
  - No selection: empty state "Select a block to edit"
  - Container block: title, slug, block_type select, sort_order
  - Leaf block: title, slug, block_type, difficulty, bloom level, estimated read time, is_published toggle + Tiptap editor for content

### Controller Methods

- `index(CanonicalTopic $topic)` — render page with topic + blocks tree
- `store(CanonicalTopic $topic)` — create new block under topic
- `update(ContentBlock $block)` — update block metadata/content
- `destroy(ContentBlock $block)` — delete block (cascades to children)
- `reorder(CanonicalTopic $topic)` — batch update sort_order + path

### Key Behavior

- `path` auto-calculated from tree position (e.g., "2.4.1.3")
- `depth_level` auto-set based on nesting depth
- `is_container` auto-set to true when block has children, can also be manually set on create
- Creating a block under a parent makes the parent a container
- Deleting a container cascades to all children

## Feature 2: Course-Block Mappings

**Page:** `/admin/courses/{course}/block-mappings`
**Controller:** `CourseBlockMappingController`
**Entry point:** "Block Mappings" dropdown menu item on courses index

### Layout

Two-panel split (similar to existing topic mappings page):

- **Left panel:** Block picker — select a topic, then browse its block tree. Click a block to add it.
- **Right panel:** Mapped blocks list with per-block config:
  - Teaching depth (Select: Surface Mention, Introductory, Intermediate, Advanced, Specialized)
  - Is core block (toggle, default true)
  - Week start / Week end (number inputs)
  - Lecture hours / Lab hours (number inputs)
  - Remove button

### Controller Methods

- `index(InstitutionCourse $course)` — render page with current mappings + available topics/blocks
- `update(InstitutionCourse $course)` — bulk save all mappings (delete-and-recreate pattern)

### Data Flow

- Controller passes published topics with block trees eager-loaded, plus existing block mappings
- Frontend sends: `{ mappings: [{ content_block_id, teaching_depth, is_core_block, week_start, week_end, lecture_hours, lab_hours }] }`
- All mappings use `institution_course_id` (not `curriculum_subject_level_id`)
- Existing topic-level mappings at `/admin/courses/{course}/mappings` remain untouched

## Feature 3: Scheme of Work Manager

**Page:** `/admin/scheme-of-work`
**Controller:** `SchemeOfWorkController`
**Entry point:** Sidebar item "Scheme of Work" under Content Management

### Layout

Top filter bar + grid view.

**Filter bar** (cascading selects):
Education System → Tier → Level → Subject → Term (1/2/3)

**Grid view** (once all filters selected):
- Table: Week #, Topic Label, Canonical Topic (searchable select), Content Block (searchable select, scoped to selected topic)
- Rows for weeks 1–13
- Inline editing per row
- "Add Week" button + "Save All" button

### Controller Methods

- `index()` — render page with education systems for first filter
- `load(Request $request)` — partial reload: given level_subject_id + term, return existing items
- `update(Request $request)` — bulk save items for level_subject + term (delete + insert)

### LevelSubject Resolution

The cascading filters resolve to a `level_subject_id`. If the level + subject + stream combination doesn't exist yet, the controller creates the `LevelSubject` record automatically (find-or-create pattern).

## Tests

### ContentBlockControllerTest (~12 tests)
- Index page renders for topic with blocks
- Store creates block with correct path and depth
- Store creates child block under parent (auto-sets parent as container)
- Update modifies block metadata
- Update modifies leaf block content (Tiptap JSON)
- Destroy deletes block and cascades to children
- Reorder updates sort_order and recalculates paths
- Validation: title required, block_type required, depth_level max 5
- Validation: leaf blocks require estimated_read_time
- Unauthenticated user redirected

### CourseBlockMappingControllerTest (~6 tests)
- Index page renders with existing mappings
- Update saves new mappings with teaching depth and week range
- Update replaces existing mappings (delete + insert)
- Validation: content_block_id required, teaching_depth valid enum
- Validation: week_end >= week_start
- Unauthenticated user redirected

### SchemeOfWorkControllerTest (~8 tests)
- Index page renders with education system filter options
- Load returns items for given level_subject + term
- Load creates LevelSubject if it doesn't exist (find-or-create)
- Update saves items with correct week numbers
- Update replaces existing items for same level_subject + term
- Validation: term 1-3, week_number 1-13, topic_label required
- Unique constraint prevents duplicate week entries
- Unauthenticated user redirected

## What We're NOT Building

- No bulk CSV import for blocks, block mappings, or scheme items
- No level subjects standalone admin page (find-or-create in scheme of work)
- No changes to existing topic CRUD (monolithic content field preserved)
- No changes to existing course-topic mappings (coexist with block mappings)
- No student-facing features
- No content block seed data (admins populate via UI)
