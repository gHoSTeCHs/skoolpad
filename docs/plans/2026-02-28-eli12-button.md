# ELI12 Button Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Let admins author a simplified version of topic/block content, and let students toggle into "Simple Mode" on the topic page and block reader with a single button.

**Architecture:** `simplified_content` is stored as a separate nullable JSON column on both `canonical_topics` and `content_blocks` (same structure as the existing `content` column â€” a Tiptap document). The admin edit form gains a second Tiptap editor. The student topic page and block reader gain a toggle button; when `simplified_content` is null the button is hidden. The toggle is pure front-end state â€” no extra API calls.

**Tech Stack:** Laravel 12, Pest 4, Inertia v2, React 19, Tiptap (existing `TiptapEditor` + `TiptapRenderer` components), Tailwind CSS v4.
